# UI/UX Audit — Wave 1 (2026-03-04)

Audit of the DotMac Omni CRM admin portal against the Industrial Modern design system.

---

## P0 — Critical (Design System Violations)

### 1. Leads page bypasses design system macros
**File:** `templates/admin/crm/leads.html`
- Stats cards are raw HTML instead of `stats_card()` macro — missing gradient icons, hover glow, domain colors.
- Header is raw HTML instead of `page_header()` — no icon badge, inconsistent spacing.
- "New Lead" button is raw HTML (`rounded-lg`, flat `bg-primary-600`) instead of `action_button()` — wrong border-radius (`rounded-lg` vs `rounded-xl`), no gradient, no shadow.
- Filter selects are raw HTML (`rounded-lg`) instead of `filter_select()` macro — wrong border-radius, missing focus ring styling.
- Cards use `rounded-xl` instead of `rounded-2xl`.
- **Impact:** Leads is a core CRM page — inconsistency here undermines the entire design system.

### 2. Tickets page uses wrong domain color
**File:** `templates/admin/tickets/index.html`
- Uses `amber/orange` for `ambient_background()` and `page_header()`.
- Design system specifies **rose/pink** for Tickets/Support domain.
- **Impact:** Breaks domain color consistency; Tickets visually blends with Customers/Subscribers.

## P1 — High (Performance & Consistency)

### 3. External Google Fonts loaded despite self-hosted fonts
**File:** `templates/base.html:20-22`
- Loads DM Sans + Outfit from `fonts.googleapis.com` even though both are self-hosted in `static/fonts/`.
- DM Sans isn't even used in the design system (Outfit + Plus Jakarta Sans are the specified fonts).
- **Impact:** Unnecessary external dependency, extra DNS lookups, GDPR risk, ~200ms added to FCP.

### 4. Topbar search input uses wrong border-radius
**File:** `templates/layouts/admin.html:159`
- Global search uses `rounded-lg` instead of `rounded-xl` per design system button/input spec.
- **Impact:** Visual inconsistency in the most prominent interactive element on every page.

### 5. Misleading code comment in base.html
**File:** `templates/base.html:19`
- Comment says "DM Sans for headings, Inter for body" but actual fonts are Outfit (display) + Plus Jakarta Sans (body).
- **Impact:** Developer confusion; could lead to wrong font references in new code.

## P2 — Medium (Polish & Minor Inconsistencies)

### 6. Toast notifications use rounded-lg
**File:** `templates/base.html:230`
- Toast badges use `rounded-lg` — should be `rounded-xl` for consistency with buttons/inputs.

### 7. Leads page filter form uses full page reload
**File:** `templates/admin/crm/leads.html:42-95`
- Traditional `<form method="get">` instead of HTMX live filtering.
- Other list pages (tickets, contacts, customers) use HTMX `hx-get` with debounced search.
- **Impact:** Slower UX on a frequently-used page.

### 8. Leads page stats cards lack icons and gradient treatment
- No SVG icons, no gradient badge, no domain color (amber/orange for CRM).
- Plain `text-2xl font-bold` metric vs `text-3xl font-bold` in `stats_card()` macro.

### 9. Dashboard imports Google Fonts DM Sans in heading CSS
- `base.html` imports DM Sans from Google, but no template uses it.
- Wasted bandwidth on every page load.
