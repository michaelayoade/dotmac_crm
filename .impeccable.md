# DotMac Omni CRM — Design Context

## Design Context

### Users
**Primary users**: Mixed office operations staff and field technicians at ISPs/telcos.
- **Office ops teams**: Managing subscribers, tickets, network, and projects from desktops during business hours. Need density and efficiency.
- **Field technicians**: On tablets and phones during installs, maintenance, and emergency repairs. Need clarity, large touch targets, and fast-loading pages.
- **Support agents**: Handling real-time customer issues via omni-channel inbox (WhatsApp, email, SMS, etc.). Need quick context switching and situational awareness.
- **Context**: Users are often multitasking, sometimes under time pressure. The tool must reduce cognitive load, not add to it.

### Brand Personality
**Technical, reliable, warm.**
- **Technical**: This is a professional operations tool. It should feel precise, capable, and engineered — not playful or consumer-grade.
- **Reliable**: Every element should communicate stability and trustworthiness. No flashy surprises. Consistent patterns that build muscle memory.
- **Warm**: Despite being technical, it should feel human and approachable. The orange accent and rounded corners soften the industrial base. Users spend all day here — it shouldn't feel cold.

**Emotional goal**: **Calm and clarity.** Even during incidents or high-load periods, the interface should reduce cognitive load and help users focus on what matters. The UI should feel like a well-organized control room, not a cluttered dashboard.

### Aesthetic Direction
**"Industrial Modern"** — the established design language. Clean, functional, with subtle sophistication:
- **Color**: Teal/cyan primary (`#06b6d4`) for trust and technical feel + warm orange accent (`#f97316`) for energy and action
- **Typography**: Outfit for display (tighter tracking, premium feel) + Plus Jakarta Sans for body (highly readable)
- **Surfaces**: Clean whites/slates with subtle gradients, noise textures, and glass morphism for depth
- **Shadows**: Layered "premium" shadows for dimensionality without heaviness
- **Motion**: Subtle, purposeful animations (fade-in-up for cards, gentle hovers) that respect `prefers-reduced-motion`
- **Dark mode**: First-class, class-based. Every color pairing must work in both modes.

**Anti-references**: Should NOT look like:
- Consumer SaaS (too playful, too many illustrations)
- Enterprise ERP from the 2000s (cluttered, gray, no visual hierarchy)
- Crypto/fintech dashboards (too dark, too neon, too much visual noise)

### Design Principles

1. **Clarity over density**: Information should be scannable. Use hierarchy (size, weight, color) to guide the eye. White space is a feature, not waste — but don't sacrifice utility for minimalism.

2. **Consistency builds trust**: Every domain has its color. Every status has its badge. Every action follows the same pattern. Users should never wonder "what does this color mean?" or "where is that button?"

3. **Responsive to context**: Office users get data-dense views. Field techs get large touch targets and simplified flows. The same information adapts to the device and situation.

4. **Calm under pressure**: During incidents or high-volume periods, the UI should help users stay focused. Avoid visual noise, excessive alerts, or competing for attention. Important information should surface naturally through hierarchy, not through urgency theater.

5. **Warm professionalism**: Technical doesn't mean sterile. Rounded corners, warm accents, thoughtful micro-interactions, and human-readable language make the tool feel approachable without sacrificing credibility.

## Design System Reference

### Fonts
- **Display** (h1-h3, metrics): Outfit — `font-display`, weights 400-800, letter-spacing -0.02em
- **Body** (text, labels): Plus Jakarta Sans — `font-body` (default), weights 400-700
- Self-hosted in `static/fonts/`

### Color System
| Domain | Primary | Secondary | Use |
|--------|---------|-----------|-----|
| Customers/Subscribers | amber | orange | Customer records, subscriber pages |
| Network/IP | cyan | blue | Network devices, IP management |
| Fiber/OLTs | violet | purple | Infrastructure, optical equipment |
| Tickets/Support | rose | pink | Support, incident management |
| Projects | emerald | teal | Project management, work orders |
| System/Settings | indigo | violet | Admin, configuration |

### CRM Channel Colors (CSS custom properties)
Email=violet, WhatsApp=green, SMS=orange, Telegram=sky, Webchat=amber, Phone=orange, Facebook=blue, Instagram=pink

### Border Radius
- `rounded-lg` (8px): Badges, small elements
- `rounded-xl` (12px): Buttons, inputs, tabs
- `rounded-2xl` (16px): Cards, containers, filter bars
- `rounded-full`: Avatars, notification dots

### Dark Mode
Class-based (`dark:` prefix). Every element needs light + dark variants. Respects `prefers-color-scheme` on first load, persists choice in `localStorage`.

### Accessibility (WCAG 2.1 AA)
- Status: icon + color (never color alone)
- Icon-only buttons: `aria-label` required
- Focus rings: `focus:ring-2 focus:ring-{color}-500/20`
- Touch targets: minimum 40px
- All animations: `prefers-reduced-motion: reduce` support

### Component Library
All reusable components in `templates/components/ui/macros.html`. Always use macros — never duplicate patterns in raw HTML.
